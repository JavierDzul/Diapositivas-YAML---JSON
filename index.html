2.3 Métodos para la detección
y correción de errores

Recapitulando

LRC, VRC, Checksum, CRC y Código Hamming

LRC (Chequeo de Redundancia Longitudinal)

  • Consiste en añadir un bit de paridad extra (o varios) al final de cada unidad de datos.
  • El bit de paridad se calcula según la paridad longitudinal de los bits en la unidad de datos.
  • Se utiliza para detectar errores de un solo bit y algunos tipos de errores en ráfaga.

VRC (Chequeo de Redundancia Vertical)

  • Involucra añadir un bit de paridad para cada posición de bit en un grupo de unidades de datos.
  • Cada bit de paridad se calcula basado en los bits en la misma posición a través de las unidades de datos.
  • Se utiliza para detectar errores en bits individuales dentro de una unidad de datos.

Checksum (Suma de Comprobación)

  • Es un valor calculado a partir del contenido de una unidad de datos (p. ej., un mensaje) usando un algoritmo matemático.
  • El checksum se añade a la unidad de datos y se transmite junto con ella.
  • En el receptor, se recalcula el checksum y, si coincide con el recibido, se considera que los datos están libres de errores.
  • Comúnmente utilizado en protocolos de red y de transferencia de archivos para la detección de errores.

CRC (Chequeo de Redundancia Cíclica)

  • Técnica avanzada de detección de errores que genera un checksum basado en la división polinómica binaria de los datos por un polinomio generador predefinido.
  • El residuo de esta operación de división se convierte en el valor CRC.
  • Ampliamente utilizado en sistemas de comunicación digital, incluyendo redes Ethernet, conexiones USB y dispositivos de almacenamiento.

Código Hamming

  • Código de detección y corrección de errores que agrega bits de paridad adicionales a una unidad de datos.
  • El número de bits de paridad se determina en función de la distancia de Hamming entre los bits de datos.
  • Permite detectar y corregir errores de un solo bit y ciertos errores de múltiples bits, útil en sistemas críticos como la memoria de computadoras.

Comparación de Métodos

Método Tipo de Error Detectado Capacidad de Corrección
LRC Errores de un solo bit y algunos errores en ráfaga No
VRC Errores en bits individuales No
Checksum Errores en el contenido completo del mensaje No
CRC Errores de múltiples bits y errores en ráfaga No (solo detección)
Código Hamming Errores de un solo bit y ciertos errores múltiples Sí (corrige errores de un solo bit)
