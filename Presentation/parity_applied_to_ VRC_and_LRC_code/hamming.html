Código Hamming

Método de corrección de errores

¿Qué es el Código Hamming?

Es un método de corrección de errores inventado en 1950 por Richard W. Hamming. Su diseño permite detectar y corregir errores simples en la transmisión de datos mediante bits de paridad.

Estructura del Código Hamming

El mensaje se divide en dos partes: los datos a transmitir y la redundancia (bits de código).

  • Los bits de paridad se intercalan con los bits de datos en posiciones específicas (potencias de 2: 1, 2, 4, 8, etc.).

Mínima Distancia

El código Hamming tiene una mínima distancia de 3, lo que permite detectar y corregir errores simples.

  • Para lograr esto, las posiciones de potencia de 2 se reservan para los bits de código, y las demás para los bits de datos.

Cálculo de Bits de Paridad

Cada bit de paridad verifica un conjunto específico de bits de datos:

  • El bit de paridad en la posición 1 cubre las posiciones 3, 5, 7, etc.
  • El bit de paridad en la posición 2 cubre las posiciones 3, 6, 7, etc.
  • Y así sucesivamente.

Cálculo del Síndrome

Durante la recepción, se recalculan los bits de paridad y se comparan con los recibidos:

  • Si el síndrome es nulo (todos ceros), no se detectan errores.
  • Si el síndrome no es nulo, marca la posición donde se produjo el error y permite corregirlo.

Políticas de Corrección

El código Hamming permite opciones de detección y corrección:

  • Puede usarse para corregir errores simples y detectar errores dobles.
  • También puede configurarse solo para detectar errores dobles sin corregir errores simples.
