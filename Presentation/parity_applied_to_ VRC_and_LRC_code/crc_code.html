El código CRC se basa en ciertas propiedades matemáticas que satisface el cociente entre polinomios.

Cyclic Redundancy Check

Agregar al patrón de bits que compone el mensaje un conjunto de bits adicionales, de manera que el patrón resultante (considerado como un polinomio binario) sea divisible de manera exacta por un polinomio denominado polinomio generador.

Cyclic Redundancy Check

Funcionamiento

Paso 1: Selección del Polinomio Generador

El primer paso es seleccionar un polinomio generador, que será usado en la división polinómica.
Ejemplo sencillo: 1 1 0 1.

\[ x^3 + x^2 + 1 = 1 1 0 1 \]

Paso 2: Expansión del Mensaje

Agrega ceros al final del mensaje en la misma longitud del polinomio generador menos uno.

  • Ejemplo: Si el mensaje original es 1001 y el polinomio generador es 1101, añade 3 ceros: 1001000.

Paso 3: Ejecución

Realiza una operacion XOR entre el mensaje expandido y el polinomio generador.

  • Continúa el proceso hasta que el residuo tenga menos bits que el polinomio generador.

\[ 0 \oplus 1 = 1 \] \[ 1 \oplus 1 = 0 \] \[ 0 \oplus 0 = 0 \] \[ 0 \oplus 1 = 1 \]

Resultado del XOR:

\[ 1001000\oplus 1101 = 101 \]

Paso 4: Obtención del Residuo

El residuo obtenido al final se convierte en el código de CRC.

  • Ejemplo: Si el residuo es 101, este será el CRC.

Paso 5: Añadir el CRC al Mensaje

El residuo (CRC) se agrega al final del mensaje original antes de enviarlo.

  • Mensaje transmitido = Mensaje original + CRC

Verificación en el Receptor

El receptor realiza la misma división binaria sobre el mensaje recibido (mensaje + CRC). Si el residuo es cero, el mensaje está intacto.

  • Residuo = 0 → Mensaje sin errores
  • Residuo ≠ 0 → Mensaje con errores

Roles del CRC

El algoritmo CRC cumple dos roles:

  • Permite determinar los bits que se deben agregar al mensaje original.
  • Permite verificar si el mensaje recibido contiene o no errores.

El receptor puede ir dividiendo el mensaje a medida que recibe los bits:

  • No es necesario recibir el mensaje completo para comenzar a verificar el CRC.
  • Los errores de ráfaga de una longitud igual o menor al grado del polinomio, SERÁN SIEMPRE DETECTADOS

Polinomios Generadores en CRC

El polinomio generador a ser usado debe ser elegido con cuidado, ya que la capacidad de detección de errores dependerá de sus características.

Existen varios polinomios actualmente en uso:

  • CRC-4-ITU: \( x^4 + x + 1 \)
  • CRC-16-IBM: \( x^{16} + x^{15} + x^2 + 1 \)
  • CRC-CCITT: \( x^{16} + x^{12} + x^5 + 1 \)
  • CRC-32: \( x^{32} + x^{26} + x^{23} + x^{22} + x^{16} + x^{12} + x^{11} + x^{10} + x^8 + x^7 + x^5 + x^4 + x^2 + x + 1 \)
